# 🎨 Canon AI Vision v5.0 UI/UX 요구사항 및 레이아웃 기획서 (White Theme)
**작성일시**: 2026년 3월 27일 14:36 (사용자 심화 피드백 최종 반영)

이 문서는 시스템의 방대한 MLOps 기능을 '단순하고 깔끔한 흰색(Clean White) 배경' 위에 어떻게 배치하고 통제할지에 대한 UI 설계 명세서입니다.

---

## 1. 🔍 TAB 1: [낮] 주간 관제실 및 병목 통제실 (Live Monitor)
실시간 영상을 단순히 띄우는 것에서 나아가, 성능과 타겟 설정을 한 화면에서 완벽히 제어합니다.

### 📱 [화면 레이아웃 구성 예시]
- **[좌측 60%] 실시간 영상 뷰어**: 가장 큰 영역을 차지. 현재 들어오는 카메라 영상을 띄우며, 그 위에 모델이 찾아낸 박스(Bounding Box)가 실시간으로 그려집니다.
- **[우측 40%] 병목 & 통계 대시보드 (KPI Widgets)**:
  1. **구간별 연산 시간(Latency) 워터폴 바**:
     - `Total Inference Time: 45ms` (숫자 표기)
     - `YOLOv8 추론(12ms)` | `CLAHE/Blur 전처리(8ms)` | `ORB 매칭(25ms)` 형태로 쪼개진 컬러 프로그레스 바로 표시하여 어디서 렉이 걸리는지 직관적으로 확인.
  2. **통계적 점수 분포 박스플롯(Box Plot)**:
     - 단순 평균 점수 외에도 최근 100프레임의 ORB 일치율 퍼센트를 박스플롯(최소범위, 1사분위, 중앙값, 3사분위, 최대범위)으로 그려 정확도의 '흔들림(분산)'을 감시.
- **[우측 하단] 타겟 ROI 마스터 편집기 (Target Editor)**:
  - 현재 검사 중인 정답(Anchor) 사진 썸네일 표시.
  - 마우스를 드래그하여 검사하고 싶은 여러 영역(ROI 1, ROI 2)을 직접 네모로 지정하고, 이들의 검사 순서를 드래그 앤 드롭 리스트로 배열.
  - `[ROI 저장 후 즉각 반영]` 버튼으로 실시간 코드 수정 없이 검사 포인트 핫리로드.

## 2. 🌙 TAB 2: [밤] 야간 학습 지시 및 사전 검수실 (Tuning Commander)
"기계가 알아서 해"가 아니라, [AI 사전 분류 ➡️ 사람의 최종 검수 및 가중치 지시 ➡️ 야간 학습] 이라는 신뢰도 100%의 워크플로우를 가집니다.

### 🤖 [Track B 데이터 통제 프로세스 & UI]
1. **AI 1차 자동 분류 (Siamese + Gemini API 작동)**
   - UI에 진입하면 사용자가 수집한 폴더나 주간에 모인 `pending` 데이터들이 어지럽게 널려있지 않습니다.
   - 이미 기계가 1차적으로 `[AI가 정상이라 확신함]`, `[제미나이 왈: 이건 애매함]`, `[AI가 아예 판단 포기함]` 의 3단 랙(Rack)으로 예쁘게 썸네일을 분류해둔 상태로 사용자를 맞이합니다.
2. **사용자 2차 최종 검수 및 가중치 패널 (Control Panel)**
   - 기계가 '이건 정상 같은데?'라고 분류한 것을 반장님이 보고 "아니야, 이건 오답이야!"라고 수동 정정할 수 있습니다.
   - **가중치 슬라이더 (Scale 1~10)**: 유독 앞으로도 자주 틀릴 것 같은 노이즈 사진을 클릭해 중요표시(Loss 가중치 폭증)를 걸어줍니다.
   - **Hard Negative 버튼**: 확실한 오답(잡음) 데이터에 붉은 직인을 찍어, 샴 네트워크가 강제로 오답을 밀어내도록 무거운 페널티 훈련 지시를 내립니다.
3. **훈련 격발**: 사용자(사람)의 완벽한 훈련 지침서 작성이 끝나면 `[🚀 야간 모델 튜닝 시작]` 버튼을 누릅니다.

## 3. 🌅 TAB 3: [아침] 결재 대시보드 및 성능 비교 (Morning Report)
어젯밤 돌려둔 학습이 아침에 얼마나 성과를 냈는지, "과거의 나"와 "현재의 나"를 비교해주는 성적표입니다.

### 📊 [비교 분석 막대 그래프 (Bar Chart Dashboard)]
UI 중앙에 큼지막한 그룹형 막대 그래프(Grouped Bar Chart)를 배치하여, **[전전 모델 v1.9] vs [어제 모델 v2.0] vs [오늘 밤 튜닝된 새 모델 v2.1]** 3개 버전을 비교합니다.

**[반드시 들어가야 할 핵심 지표 4가지]**
1. **정확도 (Recall & mAP50)**
   - 에러를 하나도 놓치지 않고 잡아낸 비율 (높을수록 좋음).
2. **연산 지연 시간 (Total Processing ms)**
   - 전처리를 너무 강하게 주느라 속도가 박살나지 않았는지 확인 (낮을수록 좋음).
3. **ORB 임계값별 매칭 정확도 (Threshold Curve)**
   - "일치도 기준점을 60%로 했을 때, 70%로 올렸을 때"의 누적 정확도가 어떻게 변했는지 보여주는 지표 (그래프).
4. **오탐률 (False Positive Rate)**
   - 정상을 비정상으로 잘못 잡아낸 횟수 (낮을수록 좋음).

**결정 버튼**: 이 지표들과 Gemini가 쓴 3줄 요약을 참고하여 `[✅ 업데이트 승인]` 또는 `[❌ 거부하고 이전 모델 롤백]` 버튼을 클릭.

## 4. ⬜ 단순 흰색(White Theme) 화면 배치 설계 전략 (Layout Blueprint)

1. **테두리(Border) 억제 및 그림자(Drop Shadow) 활용**
   - 흰 화면에 검은 선을 많이 그으면 엑셀처럼 촌스러워집니다. 각 구역은 선 대신 **옅은 회색 그림자 바탕(Elevated Card)**으로 띄워서 분리합니다.
2. **확실한 색상 규율 (Color Hierarchy)**
   - **배경 / 패널**: 퓨어 화이트(`#FFFFFF`) 및 오프 화이트(`#F8F9FA`).
   - **텍스트**: 새카만 색 대신 진한 딥그레이 `#2C3E50`.
   - **포인트 / 긍정 조작**: 맑은 파랑(`#3498DB`) 및 그린(`#27AE60`); `업데이트 승인`, `관제 정상`.
   - **가중치/위험/Hard Negative**: 명확한 주황/적색(`#E74C3C`); 집중력을 요하는 부분.
3. **넓은 여백 (Negative Space)**
   - 각 분석 지표(ms 워터폴, 막대 차트 등)를 다닥다닥 붙이지 않고 넉넉한 픽셀 마진(20px 이상)을 확보하여 화면의 숨통을 트여줍니다.
