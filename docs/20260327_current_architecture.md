# 🏭 Canon AI Vision - 현재 시스템 구조 및 구동 파이프라인 분석 문서
**작성일시**: 2026년 3월 27일 13:53 (UI 재설계를 위한 현재 상태 점검)

본 문서는 새롭게 화면(UI)을 설계하기 전, 현재 시스템 백엔드(AI 엔진)가 어떤 절차로 구동되며, 데이터가 어떻게 흘러가는지(Data Flow)에 대한 완벽한 해부도입니다.
화면의 껍데기(GUI)는 초기화되었지만, 속눈썹처럼 촘촘하게 짜여진 **핵심 AI 추론 엔진 및 자가 학습(MLOps) 파이프라인**은 아래와 같이 온전히 보존되어 작동합니다.

## 1. 🏗️ 전체 디렉터리 및 모듈 구조 (Architecture)

시스템은 크게 **[실시간 관제(Engine)]** 영역과 **[야간 자가학습(Offline)]** 영역으로 나뉩니다.

```text
머신러닝 캐논 2.1/
├── engine/ (실시간 낮 🌞 추론 엔진 - 밀리초 단위 속도전)
│   ├── preprocessor.py  : 공장 조명 보정 (CLAHE) 및 화질 선명화
│   ├── detector.py      : YOLOv8을 이용해 화면에서 모니터 패널만 잘라냄
│   ├── matcher.py       : ORB 특징점 기반 DNA 매칭 식별 알고리즘
│   ├── frame_skipper.py : 1연산 2스킵 기법으로 CPU 폭발을 막는 최적화 모듈
│   └── ocr_fallback.py  : (현재 비활성) 텍스트 기반 2차 방어막
├── offline/ (야간 달 🌙 무인 자동 라벨링 및 학습 엔진)
│   ├── siamese_classifier.py : ResNet18 기반 특징점 벡터 거리(Cosine Similarity) 분류기
│   ├── llm_judge.py          : 샴 네트워크가 헷갈려하는 사진을 클라우드 AI(GPT/Claude)에게 묻는 최종 재판관
│   └── auto_tuner.py         : 하이퍼파라미터 자동 조율기
├── scripts/ (단독 실행 유틸리티)
│   ├── video_to_frames.py : 에러난 뚱뚱한 동영상(.mp4)을 초당 5장짜리 정체 단편 사진들로 해체하는 정육점 도구
│   └── train_yolo.py      : YOLO 재학습 파이프라인 스크립트
├── data/ (데이터 창고)
│   ├── target_image/ : AI가 정답으로 알고 있는 100점짜리 기준 화면들 (Anchors)
│   ├── pending/      : 낮에 에러로 판정되어 밤에 검사받기 위해 대기중인 사진들
│   ├── labeled/      : 샴 네트워크나 LLM이 정답이라고 판단해 자동으로 라벨링이 끝난 사진들
│   └── rejected/     : AI도 포기해서 결국 사람(공장 반장)이 직접 봐야하는 완전 불량 사진들
└── gui/ (사용자 화면 - 현재 V5.0 백지 상태로 초기화됨)
    └── main_window.py
```

## 2. ⚙️ 데이터 흐름 및 구동 원리 (Data Flow & Pipeline)

시스템은 낮(현장 가동)과 밤(비가동 자가 진화) 2가지 트랙으로 나뉘어 구동됩니다. 향후 UI는 이 구동 원리를 제어하기 가장 편한 형태로 설계되어야 합니다.

### [Track A] 낮(Day): 초고속 실시간 관제 및 에러 색출 파이프라인
공장 컨베이어 벨트에 모니터가 흘러갈 때의 실시간 감시 시스템입니다. (기존 VideoThread 로직)

1. **영상 유입**: 카메라(또는 비디오 파일)로부터 1장의 프레임 사진이 들어옵니다.
2. **최적화 판독 (Frame Skipper)**:
   - 모든 프레임을 검사하면 컴퓨터가 뻗습니다. `frame_skipper.py`가 1장은 검사하도록 통과시키고, 뒤따라오는 2장은 버리며(Skip) 이전 정답을 복사해서 거짓으로 넘깁니다 (CPU 점유율 방어).
3. **전처리 (Preprocessor)**:
   - 공장의 강한 역광으로 인해 모니터가 하얗게 뜰 경우, `CLAHE` 알고리즘이 자동으로 밝기를 억누르고 명암비를 극대화합니다.
4. **패널 타겟팅 (YOLO Detector)**:
   - 잡다한 배경(공장 바닥, 사람 손)을 무시하고, 모니터 화면(베젤 내부)만 정확하게 칼로 잘라냅니다(Crop).
5. **특징점 대조 (ORB Matcher)**:
   - 잘려진 모니터 화면에서 수많은 특징점(수학적 DNA 좌표)을 추출한 뒤, `data/target_image/`에 있는 정답 사진의 특징점 배열과 겹쳐봅니다. (Lowe's Ratio 적용)
6. **결과 처리**:
   - 일치도가 임계값(`MATCH_THRESHOLD = 60%`) 이상이면 **[정상(Pass)]** 처리됩니다.
   - 일치도가 60% 미만이면 **[에러(Fail)]** 판정을 내리고 즉각 해당 프레임들을 캡처하여 `data/pending/` 폴더로 던져버립니다.

> **🚨 [핵심 설계 원칙] 무조건 '안 잘린 100% 원본 프레임'을 저장할 것!**
> - YOLO가 박스를 성공적으로 찾았더라도, `pending/` 폴더에 저장할 때는 모니터 외곽(공장 배경)이 모두 포함된 1920x1080 형태의 **생 원본 영상**을 캡처해야 합니다.
> - 이유: 크롭된 결과물만 저장하면 밤에 판단을 뒤집고 YOLO를 더 맵게 재학습시킬 때, 주변 배경 정보가 날아가버려 재학습(Data-Centric AI)이 원천 불가능해지기 때문입니다.

### [Track B] 밤(Night): 무인 자동 라벨링 및 지능 진화 파이프라인 (MLOps)
GUI에서 사용자가 "학습(Train)" 버튼을 누르거나, 야간에 스케줄러가 데몬으로 돌게 되는 백그라운드 재판 과정입니다.

1. **동영상 해체기 가동 (video_to_frames.py)**:
   - 드래그 앤 드롭으로 던져진 무거운 에러 영상(.mp4)을 초당 최소 프레임(FPS 5)으로 잘라서 낱장의 `.jpg`로 쪼개 `data/pending/` 보관함에 넣습니다.
2. **1차 AI 재판: 샴 네트워크 (siamese_classifier.py)**:
   - 엣지 PC에서도 가볍게 돌아가는 `ResNet18` 대뇌가 `pending` 폴더의 불량 사진들을 들여다봅니다.
   - 사진을 512자리의 벡터(숫자)로 변환한 후, 정답 사진과의 '코사인 유사도(거리)'를 계산합니다.
   - **90점 이상**: "아 이건 100% 정상인데 조명때문에 낮에 헷갈린거네." ➡️ 사람 몰래 자동으로 정답 딱지를 붙여 `data/labeled/` 폴더로 넣어버립니다. (유지보수비용 0원 달성)
   - **50점 미만**: "이건 모니터가 아니라 사람 손이 가린거라 완전 불량임." ➡️ `data/rejected/` 사람에게 알림.
3. **2차 AI 재판: 비전 LLM (llm_judge.py)**:
   - 샴 네트워크에서 점수가 [50점 ~ 90점] 사이가 나와서 "아 기계인 내 머리로는 도저히 모르겠다" 싶은 애매한 사진들만 추려냅니다.
   - 사진을 문자열(Base64)로 압축해 보안 상태로 클라우드의 GPT-4V/Claude 등에 전송하여, "의미론적"으로 정상 화면인지 결재를 받습니다. (비용 최소화, 억까 방어)
4. **재학습 장전 (train_yolo.py)**:
   - 이렇게 자동으로 쌓여버린 `labeled/` 안의 엄청난 빅데이터들을 긁어모아 YOLO를 재학습시킵니다. 내일 낮에는 AI가 더 똑똑해져서 헷갈리지 않게 됩니다.

## 3. 🎯 향후 UI 디자인을 위한 시사점
위의 로직을 조종하기 위해, **새로운 UI는 반드시 다음 3가지 핵심 트리거(기능)를 보유하고 있어야 합니다.**

1. **관제 뷰 (낮 감시용)**
   - `cv2.VideoCapture` 흐름을 띄워줄 패널이 최소 1개 이상 필요.
   - 일치도 퍼센트(Score)가 60% 밑으로 떨어질 때 UI 전체가 빨간색으로 시각적 경고를 줄 수 있어야 함.
2. **트러블슈팅/학습 뷰 (밤 제어용)**
   - 에러가 난 동영상을 사람이 끌어다 놓을 수 있는 "드래그 앤 드롭 존" 필요 (여기서 `video_to_frames` + `siamese_classifier` 트리거).
   - "자동 AI 재학습 시작" 이라는 큼지막한 액션 버튼 장착 필수.
3. **상태 알림판 (심리적 안정감)**
   - 백그라운드에서 AI가 스스로 고민하고 자동 할당한 데이터 건수(Labeled, Rejected 등) 통계를 띄워줄 단순한 막대기나 숫자가 있으면 좋음.

## 4. 🚀 추가 고도화 및 파이프라인 전면 개편 계획 (To-Do)
본 시스템을 완벽한 상용화 MLOps 플랫폼으로 거듭나게 하기 위해, 아래의 4가지 파멸적(파괴적) 개선안이 설계 및 적용(일부) 되었습니다.

### ① Track B의 VLM을 Google `Gemini API`로 교체 완료
- **변경 사항**: 기존 `llm_judge.py`를 완전히 뜯어고쳐, 사용자의 API Key(`AIza...`)를 바탕으로 초고속 비전 처리가 가능한 구글 **제미나이 1.5 (gemini-1.5-flash)** REST API 모듈로 업그레이드 하였습니다. (속도 및 정확도 압도적 상향).

### ② 자동 파라미터 최적화 (Hyperparameter Tuning - `auto_tuner.py`)
- **설계 구조**: 단순 5가지 필터를 수동 온오프 하는 것을 넘어, 내부 수치(예: CLAHE clipLimit 2.0 vs 3.0)를 자동으로 찾는 기능.
- **계층적 탐색(Coarse-to-Fine)**:
  - Step 1: 대표값 32조합 빠른 전수 테스트.
  - Step 2: 살아남은 상위 조합들을 **`Optuna` 베이시안 최적화(Bayesian Optimization)** 알고리즘을 사용해 가장 효율적으로 정밀 타격(Fine-Tuning)하여 시간을 단축시키는 로직이 추가될 것입니다.

### ③ 사용자 주도 훈련 통제 (데이터 가중치 & Hard Negative 활용)
- **중요도 부여(Scale 1~10)**: 사용자가 UI에서 유독 잡히지 않는 노이즈 에러 사진에 강력한 점수를 매기면, 야간 훈련 시 해당 사진의 **Loss(가중 손실 함수)**를 뻥튀기시켜 모델 뇌에 박아버립니다. (의도적 과적합 기법)
- **오답 노트(Hard Negative) 시스템**: 삭제된 오염 데이터를 단순히 쓰레기통에 버리지 않고, 샴 네트워크에게 "절대 정답이 아니니 속지 말라"고 단련시키는 가혹 훈련 재료로 100% 재활용합니다.

### ④ 롤백(Rollback) 방어막 & 성적표(Dashboard) 시스템
- **모델 스냅샷**: 야간 진화가 어긋났을 때를 대비하여 파일명(e.g., `yolov8n_20260327.pt`)으로 버전 관리를 진행, UI 버튼 하나로 어제 버전 뇌(모델)로 즉각 탈착 복구하는 기능을 추가합니다.
- **성능 시각화**: 어제 훈련과 오늘 훈련 후의 Recall, mAP50, FPS 추락/상승세를 그래프로 시각화하여, 공장 반장님이 한눈에 "아 오늘은 더 잘 잡는구나" 판단할 수 있게 구성합니다.
