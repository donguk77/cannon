# 캐논 AI 비전 시스템 최적화 및 정확도 개선 계획

본 문서는 현재 시스템에서 발견된 성능 저하(FPS) 및 탐지 정확도(ORB 점수) 문제를 해결하기 위한 구체적인 분석 결과와 향후 구현 계획을 정리합니다.

---

## 1. 실시간 FPS 저하 문제 분석 및 최적화
### 🔍 원인 분석 (Problem)
- **고정 대기 로직 존재**: `gui/tab_monitor.py` 내 `VideoThread` 루프 하단에 `msleep(delay)`가 존재하며, 여기서 `delay`는 영상의 원본 FPS(30)에 맞춰 **33ms로 고정**되어 있음.
- **연쇄 지연**: AI 연산(YOLO+ORB)에 77ms가 소요될 경우, 여기에 강제 대기 33ms가 더해져 프레임당 총 110ms가 소요됨. 이로 인해 실제 처리 속도는 $1000/110 \approx$ **9 FPS**로 제한됨.

### 🛠️ 개선 계획 (Solution)
- **적응형 수면(Adaptive Sleep) 도입**:
  - 연산 시간이 목표 프레임 타임(33ms)을 초과할 경우 대기 시간을 0 또는 최소화(1ms)로 자동 조절.
  - 연동 공식: `sleep_time = max(1, target_delay - processing_time)`
  - 이를 통해 연산 속도(77ms)에 정비례하는 **13 FPS**를 확보함.

## 2. ORB 매칭 정확도(점수) 개선
### 🔍 원인 분석 (Problem)
- **비교 대상의 불일치(Scale/ROI Mismatch)**:
  - **실시간 화면**: YOLO 모델이 모니터 베젤(알맹이)만 정확히 잘라낸 **'크롭된 이미지'**를 사용.
  - **정답 이미지(1.png~4.png)**: 윈도우 작업표시줄, 바탕화면 등이 모두 포함된 **'스크린샷 전체'**를 사용.
- **결과**: AI가 '부분'과 '전체'를 대조하려니 특징점 위치가 어긋나 동일 이미지임에도 점수가 10점 미만으로 낮게 측정됨.

### 🛠️ 개선 계획 (Solution)
- **ROI 기반 타겟 로딩**:
  - `engine/matcher.py`의 `load_targets_from_dir` 함수가 `data/roi_config.json`을 읽도록 수정.
  - 정답 이미지를 메모리에 올릴 때, 사용자가 설정한 ROI 영역으로 **먼저 크롭한 뒤** 특징점(DNA)을 추출.
  - '크롭된 실시간 화면' vs '크롭된 정답지' 구도로 변경하여 매칭 점수를 **90점 이상**으로 정상화.

---

## 3. YOLO 시각화 고도화
### 🛠️ 구현 계획 (Solution)
- **신뢰도(Confidence) 표기**:
  - `engine/detector.py`가 탐지 좌표뿐만 아니라 확신도(conf)를 함께 반환하도록 수정.
  - 화면 상단 "Bezel (YOLO)" 명찰 옆에 **신뢰도 %** (예: Bezel 98.2%)를 실시간으로 출력하여 모델의 판단 근거를 시각화함.

## 4. 시스템 안정화 및 오류 복구
- **구문 오류 복구**: 최적화 로직 적용 중 발생한 `gui/tab_monitor.py` 내의 오타 및 비정상적인 코드 구조를 완전히 복구하고 안정화함.
